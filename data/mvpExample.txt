Part 1: MVP example, step by step (what you’re building)
Scenario
User (founder) says:
“Follow up Investor X tomorrow at 8am. Keep it short. Don’t sound needy.”
Assume the external orchestrator already exists and sends you one atom (task) to process.
Step 0: Orchestrator creates an atom (external, not you)
{
"atom_id":"a_001",
"workspace_id":"w_123",
"actor_id":"u_9",
"intent_hint":"follow_up",
"goal":"Follow up Investor X tomorrow at 8am. Keep it short. Don’t sound needy.",
"entities": {
"person":"Investor X",
"time":"2026-02-25T08:00:00+05:30"
},
"connected_tools": ["gmail","calendar"]
}
Your Brain receives this atom and runs:
R → J → D → (Orchestrator executes) → L
Layer 1: Retrieval (R)
What it does
It compiles a ContextBundle so the Brain doesn’t guess.
What it fetches (MVP)
Scope: workspace, actor permissions, agent permissions
Memory: user tone preferences, follow-up cadence, investor metadata
Tool state: Gmail thread state, last email, last reply
Policies: “VIP investors require approval”, “no attachments”
Precedents: best-performing follow-up templates historically
Output: ContextBundle (simplified sample)
{
"scope_context": {
"actor_role":"founder",
"permissions": ["draft_email","schedule_send"],
"workspace_mode":"fundraising"
},

Layer 2: Judging (J)
This is scoring + flags only.
J1: Sufficiency
Do we have enough info?
recipient identified
✅
thread exists ✅
✅
time specified
So: no questions.
MVP Example
2J2: Policy compliance
VIP rule triggers → needs approval.
J3: Risk
Email to VIP investor is high reputational risk + irreversible send action → high.
J4: Priority
Fundraising mode + follow-up delay of 11 days → high.
J5: Multi-factor
Top factors: VIP, fundraising mode, no reply, user tone preference.

Layer 3: Decision (D)
Decision layer creates the DecisionPacket to send back to orchestrator.
D1: Intent type
follow_up
D2: Plan generation
It uses a template + constraints:
draft email
show to user for approval
schedule send at 8am
D3: Execution mode
needs_approval (because policy + risk)
D4: Trace
why this plan, policies applied, top factors
D5: Packaging
UI-ready response + tool call drafts (not executed)
Output: DecisionPacket (simplified)
{
"intent_type":"follow_up",
